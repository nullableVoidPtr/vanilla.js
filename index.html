Why Vanilla?

Created by Brendan Eich in 1995, Vanilla is now a thriving JavaScript library carefully crafted by our design committee ensuring that developing anything with this library (from websites to command line tools) is a simple, yet pleasant experience, as your options include (but are not limited to):

  • accessible module imports and exports
  • a comprehensive low-overhead DOM API
  • an intuitive object-oriented system for custom elements
  • a handy library for processing arrays
  • an highly efficient Regex engine
  • AJAX functionality

Compatible with any ecosystem

As a first-class citizen of the web, Vanilla does not extend any standard object, let alone the DOM; no monkey-patching in order to see things working, no external technical debt left at the last minute, no lost productivity in exchange. Learn Vanilla once; use it anywhere.

Super lightweight

Our team of developers ensure that every byte of code is essential, as we pride in optimizing our data footprint, so that your project is only your code, and nothing else. The days of an external 3MB library dwarfing the index.js are simply over. You can even tweet the unminified embedded library in its entirety:

Blazingly fast

Simply put, we run circles around jQuery. Accessing a document's elements has never been faster, and you don't even have to sacrifice code clarity for speed.
